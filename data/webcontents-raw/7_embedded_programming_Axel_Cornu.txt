<h2 id="reading-the-datasheet">Reading the datasheet<a class="headerlink" href="#reading-the-datasheet" title="Permanent link">&para;</a></h2>
<p>I read the ATtiny24/44/84 datasheet because I used an ATtiny84 to make my HelloBoard.</p>
<p>The most interesting documentation I found in this datasheet is about the clock system (chap. 6) :</p>
<p><img alt="Clock distribution" src="../images/week09/Clock.png" title="Clock distribution" /></p>
<p>In the clock distribution diagram, you can see that there is only one <strong>clk I/O</strong> to be used for timers, interrupts or PWM.</p>
<p>As I often need to use timers for triggering interrupts, I want to know more about it&hellip;</p>
<p>In chapter 11 and 12, I found the diagrams of the timers/counters. Here is the 8-bit one :</p>
<p><img alt="Timer/Counter0" src="../images/week09/Block0.png" title="Timer/Counter0" /></p>
<p>To make it easier to read, I looked for the register&rsquo;s names :</p>
<ul>
<li>TCNT means Timer/Counter (the register that increments with time)</li>
<li>OCR means Output Compare Register</li>
<li>TCCR means Timer/Counter Control Register</li>
<li>OC means Output Compare (it can be a signal for interrupt request, or a physical output pin)</li>
<li>TOV means Timer/Counter Overflow flag</li>
</ul>
<p>And here is the 16-bit one :</p>
<p><img alt="Timer/Counter1" src="../images/week09/Block1.png" title="Timer/Counter1" /></p>
<p>The only difference in these diagrams is that Timer/Counter1 has a <strong>ICRn</strong> (Input Capture Unit), and I don&rsquo;t know what it&rsquo;s meant for.</p>
<p>I also found the diagram explaining the <strong>Output Compare Unit</strong> :</p>
<p><img alt="Output Compare Unit" src="../images/week09/Compare.png" title="Output Compare Unit" /></p>
<p>(16-bit version&rsquo;s diagram is so complicated that I don&rsquo;t want to display it here.)</p>
<p>In this diagram :</p>
<ul>
<li>FOC means Force Output Compare</li>
<li>OCF means Output Compare Flag</li>
<li>WGM means Waveform Generator Match</li>
<li>COM means Compare Output Mode</li>
</ul>
<p>In subscripts, <strong>n</strong> is for the number of the Timer/Counter (0 or 1) and <strong>x</strong> is for an identification letter (A or B).</p>
<p>In chapter 13, a diagram shows how a prescaler divides the I/O clock frequency for each Timer/Counter, according how you set the <strong>Clock Select</strong> register :</p>
<p><img alt="Prescaler" src="../images/week09/Prescaler.png" title="Prescaler" /></p>
<p>Now, I won&rsquo;t go into further details because I am mostly interested in controlling interrupts.
In chapter 9, I found the <strong>Interrupt Vectors</strong> table :</p>
<p><img alt="Interrupt Vectors" src="../images/week09/Vectors.png" title="Interrupt Vectors" /></p>
<p>It shows the 7 interrupt vectors from Timer/Counter0 (TIM0_&hellip;) and Timer/Counter1 (TIM1_&hellip;).</p>
<p>The names and definitions are explicit enough for me to understand what each vector is related to,
but it doesn&rsquo;t tell the link between the Timer/Counter registers and the interrupt vectors.</p>
<p>And I could not find a clear explanation in the datasheet.</p>
<h3 id="what-i-dont-understand">What I don&rsquo;t understand<a class="headerlink" href="#what-i-dont-understand" title="Permanent link">&para;</a></h3>
<ul>
<li>What&rsquo;s the use of an Input Capture Unit ?</li>
<li>what&rsquo;s the use of a Compare Match Output Unit ?</li>
<li>Why is timing setup so complicated ? It would need some kind of flow-chart to guide you into the process.</li>
</ul>
<h3 id="what-i-learned">What I learned<a class="headerlink" href="#what-i-learned" title="Permanent link">&para;</a></h3>
<ul>
<li>Each timer/counter can have different prescaler settings.</li>
<li>Each timer/counter has two independent output compare units.</li>
<li>The timer/counter1 allows 16-bit PWM (it can be useful for controlling a servo with better precision).</li>
</ul>
<h2 id="programming-with-arduino-ide">Programming with Arduino IDE<a class="headerlink" href="#programming-with-arduino-ide" title="Permanent link">&para;</a></h2>
<p>I did it to test my HelloBoard in the <a href="http://fab.academany.org/2019/labs/ulb/students/axel-cornu/assignments/week07.html#testing-with-arduino-ide">week7 assignment</a>.</p>
<h2 id="programming-with-winavr">Programming with WinAVR<a class="headerlink" href="#programming-with-winavr" title="Permanent link">&para;</a></h2>
<h3 id="makefile">Makefile<a class="headerlink" href="#makefile" title="Permanent link">&para;</a></h3>
<p>I first edited the <strong>.make</strong> files to make it work with the ATtiny84. I had to rename it <strong>Makefile</strong> so it can work with WinAVR.</p>
<p>This is how it looks like for the <a href="http://academy.cba.mit.edu/classes/embedded_programming/hello.ftdi.44.echo.c">hello.ftdi.44.echo.c</a> example :</p>
<div class="codehilite"><pre><span></span><span class="nv">PROJECT</span><span class="o">=</span>hello.ftdi.44.echo
<span class="nv">SOURCES</span><span class="o">=</span><span class="k">$(</span>PROJECT<span class="k">)</span>.c
<span class="nv">MMCU</span><span class="o">=</span>attiny84
<span class="nv">F_CPU</span> <span class="o">=</span> <span class="m">20000000</span>

<span class="nv">CFLAGS</span><span class="o">=</span>-mmcu<span class="o">=</span><span class="k">$(</span>MMCU<span class="k">)</span> -Wall -Os -DF_CPU<span class="o">=</span><span class="k">$(</span>F_CPU<span class="k">)</span>

<span class="nf">$(PROJECT).hex</span><span class="o">:</span> <span class="k">$(</span><span class="nv">PROJECT</span><span class="k">)</span>.<span class="n">out</span>
    avr-objcopy -O ihex <span class="k">$(</span>PROJECT<span class="k">)</span>.out <span class="k">$(</span>PROJECT<span class="k">)</span>.c.hex<span class="p">;</span><span class="se">\</span>
    avr-size --mcu<span class="o">=</span><span class="k">$(</span>MMCU<span class="k">)</span> --format<span class="o">=</span>avr <span class="k">$(</span>PROJECT<span class="k">)</span>.out

<span class="nf">$(PROJECT).out</span><span class="o">:</span> <span class="k">$(</span><span class="nv">SOURCES</span><span class="k">)</span>
    avr-gcc <span class="k">$(</span>CFLAGS<span class="k">)</span> -I./ -o <span class="k">$(</span>PROJECT<span class="k">)</span>.out <span class="k">$(</span>SOURCES<span class="k">)</span>

<span class="nf">flash</span><span class="o">:</span> <span class="k">$(</span><span class="nv">PROJECT</span><span class="k">)</span>.<span class="n">hex</span>
    avrdude -p t84 -P usb -c usbtiny -U flash:w:<span class="k">$(</span>PROJECT<span class="k">)</span>.c.hex

<span class="nf">fuses</span><span class="o">:</span> <span class="k">$(</span><span class="nv">PROJECT</span><span class="k">)</span>.<span class="n">hex</span>
    avrdude -p t84 -P usb -c usbtiny -U lfuse:w:0x5E:m
</pre></div>


<p>I removed the commands that are not related to the USBtinyISP programmer. I renamed <strong>program-usbtiny</strong> and <strong>program-usbtiny-fuses</strong> to make it shorter.</p>
<p>After editing the C code, I have to open the command prompt, use <strong>cd</strong> to go into the firmware directory, and type these commands :</p>
<ul>
<li><strong>make</strong> to build the <strong>.hex</strong> and <strong>.out</strong> files</li>
<li><strong>flash</strong> to send it to the programmer</li>
<li><strong>fuses</strong> to set the µcontroller fuses</li>
</ul>
<h3 id="c-program">C program<a class="headerlink" href="#c-program" title="Permanent link">&para;</a></h3>
<p>I tried to edit the hello.ftdi.44 C codes to make it :</p>
<ul class="task-list">
<li class="task-list-item"><label class="task-list-control"><input type="checkbox" disabled checked/><span class="task-list-indicator"></span></label> Light a different LED when receiving a character 0 or 1 on serial port</li>
<li class="task-list-item"><label class="task-list-control"><input type="checkbox" disabled checked/><span class="task-list-indicator"></span></label> Print the status of the push-button input on serial port</li>
<li class="task-list-item"><label class="task-list-control"><input type="checkbox" disabled checked/><span class="task-list-indicator"></span></label> Use the INT0 interupt to switch from one LED to another</li>
<li class="task-list-item"><label class="task-list-control"><input type="checkbox" disabled checked/><span class="task-list-indicator"></span></label> Use the INT0 interupt to print on serial port when the button is pushed</li>
</ul>
<p>The last step is tricky because the serial port input also uses interupts.
In fact, the push-button interupt function only shows up when the other interupt is set by receiving data on the serial port input.</p>
<p>I think it&rsquo;s because of the <strong>get_char</strong> function that blocks everything when doing <strong>while(pin_test(*pins,pin))</strong>.</p>
<p>With the <strong>while</strong> removed, the function still receives data from the ftdi but before that, it shows up an empty character.</p>
<p>So I had to <em>filter</em> it by adding a condition in the <strong>ISR(PCINT0_vect)</strong> function. The condition is that the received byte must be greater than 32 (first of readables characters).</p>
<h2 id="assembler-programming">Assembler programming<a class="headerlink" href="#assembler-programming" title="Permanent link">&para;</a></h2>
<p>In order to try compiling an assembler code, I am reading this <a href="http://www.avr-asm-tutorial.net/avr_en/micro_beginner/index.html">micro beginner course</a>.</p>
<p>But I don&rsquo;t want to use AtmelStudio so I had to download <a href="http://www.avr-asm-tutorial.net/gavrasm/index_en.html">gavrasm</a> as a compiler.</p>
<h3 id="unzip-gavrasm">Unzip gavrasm<a class="headerlink" href="#unzip-gavrasm" title="Permanent link">&para;</a></h3>
<p>I downloaded the executable Windows-64-Bit version 4.1 and extracted the files in C:\gavrasm (there are 3 files but only <strong>gavrasm.exe</strong> is necessary).</p>
<h3 id="makefile_1">Makefile<a class="headerlink" href="#makefile_1" title="Permanent link">&para;</a></h3>
