<h1 id="7-electronics-design">7. Electronics Design<a class="headerlink" href="#7-electronics-design" title="Permanent link">&para;</a></h1>
<p><strong>Assignment</strong> redraw an <a href="http://academy.cba.mit.edu/classes/embedded_programming/index.html#echo">echo hello-world board</a>, add (at least) a <a href="http://search.digikey.com/scripts/DkSearch/dksus.dll?Detail&amp;name=SW262CT-ND">button</a> and LED (with current-limiting resistor) check the design rules, make it, and test that it can communicate
extra credit: simulate its operation</p>
<h2 id="downloading-kicad">Downloading KiCad<a class="headerlink" href="#downloading-kicad" title="Permanent link">&para;</a></h2>
<p>I went to the <a href="https://www.kicad.org/download/">KiCad</a> website and downloaded the worldwide OSDN versio for windows. To get more familiar with the software I watched tutorial videos provided by <a href="https://app.techexplorations.com/courses/kicad-like-a-pro-3e/">Tech Explorations</a></p>
<h2 id="single-sided-pcb-in-kicad">Single Sided PCB in KiCad<a class="headerlink" href="#single-sided-pcb-in-kicad" title="Permanent link">&para;</a></h2>
<p>My plan was to create a board with a button that didn&rsquo;t act as a kill switch but as an input. The plan was for the LED to stay on unless the button was pushed.</p>
<h3 id="idea">Idea<a class="headerlink" href="#idea" title="Permanent link">&para;</a></h3>
<p>I used my <a href="../week05/">earlier documentation</a> on my blinky board as a reference for this. Except this week I was tasked to add a button. The button couldn’t act as a kill switch but as an input for the ATTiny412. The components I would use from the previous board would be:</p>
<ul>
<li>ATTiny412</li>
<li>4 Pin Button</li>
<li>LED</li>
<li>330 Ohm Resistor</li>
<li>3 Pin Header</li>
<li>1 UF Capacitor</li>
</ul>
<h3 id="kicad-schematic-editor">KiCad Schematic Editor<a class="headerlink" href="#kicad-schematic-editor" title="Permanent link">&para;</a></h3>
<p>I first had to download the component library for Fab Academy first. I used this <a href="https://gitlab.fabcloud.org/pub/libraries/electronics/kicad">GitLab</a> link to download the library. Once I downloaded the zip, I extracted the files and loaded it into a separate directory. To incorporate it into KiCad. I performed the following steps:</p>
<ol>
<li>Preferences &gt; Symbol Library &gt; Select <strong>fab.kicad_sym</strong>. This uploaded the symbols</li>
<li>Preferences &gt; Manage Footprint Library &gt; <strong>Select the directory labled fab.pretty</strong></li>
<li>Preferences &gt; Configure path &gt; Select directory where the files are downloaded</li>
<li><strong>Don&rsquo;t change the nickname of the file</strong></li>
</ol>
<p>I then started adding the symbols into the schematic editor. Most of the symbols already came with footprints.  I added the following, most of the component are from the Fab library</p>
<ul>
<li>ATTiny412 chip</li>
<li>Unpolarized Capacitor</li>
<li>Normal Resistor</li>
<li>LED</li>
<li>Male Connector</li>
<li>4 Pin Button</li>
</ul>
<p>I connected these components together. Important things to note, the power and ground had to go from the header to the capacitor then to the microcontroller. This helps the microcontroller not reset if there is a dip in power. I then drew another wire from the power wire into the LED then into the resistor then into the pin. For the button, I had to connect one wire to the ground and the other end to the ground. The last step was the UPDI, and I had one wire go directly from the pin to the UPDI pin. The next step was to edit the microcontroller symbol so I wouldn’t receive any errors after the ERC. I did this by</p>
<p>For pins</p>
<ol>
<li>Select on the pin to  </li>
<li>Electrical type: Unconnected</li>
<li>Select Invisible</li>
</ol>
<p>For the Power and Ground</p>
<ol>
<li>Electrical type power input</li>
</ol>
<p>Save and close</p>
<p><img alt="image" src="../../images/week07/WorkingSymbol_6_11zon.jpg" /></p>
<p>The microcontroller should save the edit but to make sure, I ran the ERC but there was still one error regarding the power and ground. This occurred because the ERC couldn’t find a power source such as a battery. Mr. Dubick informed us that we had to add power flags to the vcc and ground to correct this error. I saved this schematic and uploaded it into the PCB editor.</p>
<h3 id="kicad-pcb-editor">KiCad PCB Editor<a class="headerlink" href="#kicad-pcb-editor" title="Permanent link">&para;</a></h3>
<p>This process was simple but was difficult.  I simply had to connect wires to the different components. The difficulties I experienced were placing the components in the right area and not having the wires overlap or be too close together. To help me through this I created constraints for the tracks by performing the following</p>
<p><img alt="image" src="../../images/week07/Screenshot%202022-03-02%20205256_1_11zon.jpg" /><figcaption>The schematic right after uploading into PCB editor</figcaption></p>
<p>File &gt; board setup &gt; design rules &gt; constraints and I made <strong>minimum clearance 0.4 mm</strong> and the <strong>minimum track width 0.5 mm</strong></p>
<p><img alt="image" src="../../images/week07/boardSetup_8_11zon.jpg" /></p>
<p>At multiple points throughout the process, I had to change the pin of the button so that it wouldn’t cross over any wires. I went through multiple designs, but I was able to figure out one that worked. One crucial thing that I did was <strong>label the power, ground and UPDI pins on the microcontroller and the pins</strong>. This made it much easier to connect the right pins.  </p>
<p>To check that my design was good I selected the ERC and the only errors I received were silkscreen errors which were nothing to work about.</p>
<p><img alt="image" src="../../images/week07/WorkingSchematic_5_11zon.jpg" /><figcaption>My working schematic</figcaption></p>
<h3 id="exporting">Exporting<a class="headerlink" href="#exporting" title="Permanent link">&para;</a></h3>
<p>I checked that the PCB was ready to export by</p>
<ul>
<li>Making sure all of the components were on the F.Cu or the <em>front copper layer</em></li>
<li>Creating a box around my PCB on the Edge.Cuts layer.</li>
<li>Check the 3D Viewer</li>
</ul>
<p>To export the file I had to select <strong>File &gt; Plot &gt; Select the layers to plot (Usually the F.Cu and Edge. Cuts) &gt; Plot</strong>.</p>
<p><img alt="image" src="../../images/week07/SETTINGS_2_11zon.jpg" /></p>
<p>The files showed up in the directory where I saved the schematic and the pcb design.</p>
<p><img alt="image" src="../../images/week07/ShowingGbrFile_4_11zon.jpg" /></p>
<h3 id="milling">Milling<a class="headerlink" href="#milling" title="Permanent link">&para;</a></h3>
<p>I uploaded the files into bantam tools milled out the board using my <a href="../week05/">earlier documentation</a> to mill the board. Once I hit mill all visable my board was complete.</p>
<video width="700" controls>
  <source src="../../images/week07\videos\Milling_Blinky.mp4" type="video/mp4" />
</video>

<p><img alt="image" src="../../images/week07/MilledPCB_11zon.jpg" /></p>
<p><img alt="image" src="../../images/week07/FullySolderedPCB_11zon.jpg" /></p>
<h3 id="coding">Coding<a class="headerlink" href="#coding" title="Permanent link">&para;</a></h3>
<p>I used example code from <a href="https://fabacademy.org/2021/labs/charlotte/students/graham-smith/assignments/4ElectronicProduction/">Graham Smith&rsquo;s documentation</a>. I wanted the LED to stay on constantly but only turn off when I pressed the button. I made the pin the button was connected act as a pull up resistor. I then made the pin that the LED was connected to stay LOW (so the LED would turn on) unless the pin that the button was connected to receive a LOW signal (the button is pressed) which would make the LED pin run high (or stop the flow of power). I edited the pin numbers but an error I received was that the code couldn’t run with an ATTiny412 chip.</p>
<p><img alt="image" src="../../images/week07/errorforcomplingATTinyFull_10_11zon.jpg" /></p>
<p>I realized that the error was because my pins were wrong. In the code I had defined a pin as “7” but when looking at the pinout of the microcontroller, pin 7 doesn’t exists, it’s only a label. The real pin was pin 4. I changed this and the code compressed.</p>
<p>Here is the code</p>
<div class="codehilite"><pre><span></span><code>int ledPin = 1;

void setup()
{
    pinMode(ledPin, OUTPUT);  // Defines the LED pin
    pinMode(4, INPUT_PULLUP); // Makes the button pin read as HIGH if button isn&#39;t pushed
}

void loop()
{
    if (digitalRead(4) == HIGH) // Button Isn&#39;t Pushed
    {
        digitalWrite(ledPin, LOW); // LED will stay on
    }
    else
    {
        digitalWrite(ledPin, HIGH); // LED turns off
    }
}
</code></pre></div>

<p>I plugged in <a href="../week05/">my programmer</a> into the PCB, connecting the power, ground, and UPDI. I configured the tools in arduino using my <a href="../week05/">earlier documentation</a> on the ATTiny412 blinky. I ran the code and my board worked.</p>

<h2 id="double-sided-pcb-in-kicad">Double Sided PCB in KiCad<a class="headerlink" href="#double-sided-pcb-in-kicad" title="Permanent link">&para;</a></h2>
<p>This board is based off of a future project. A 7-Segment digital clock that encoporates two bluetooth speakers. Something for my senior year before I graduate from Charlotte Latin. The code I was using was built for an Arduino so I used an AtMega328P for my chip. My materials list is below</p>
<ul>
<li>AtMega328P</li>
<li>Red LED 2x</li>
<li>330 Ohm Resistor 2x</li>
<li>1 uF Capacitor</li>
<li>2 Pin Header 4x</li>
<li>3 Pin Header 3x</li>
<li>4 Pin Header</li>
</ul>
<h3 id="kicad-schematic-editor_1">KiCad Schematic Editor<a class="headerlink" href="#kicad-schematic-editor_1" title="Permanent link">&para;</a></h3>
<p>My goal for this board was to have everything run through it, PWR, GND, Data, and PGR. With this in mind I decided that each component should have their respective PWR and GND headers, and not just having ground headers and power headers connected creating a wire mess. I realized this while finishing my <a href="../../projects/final-project/">final project</a>.</p>
<p>Following similar steps to my single sided PCB documentation above, I created a power LED indicator and I also create a small programming LED so I can test some simple code. Then I added the capacitor and connected that to the main PWR, GND, and UPDI line. Finally I connected all of the pins and to their repsective component headers whether it was I2C or FTDI or the Neopixels I was using. For a double sided PCBs the schematic editor doesn&rsquo;t really change, its just most of the planning that goes into the design.</p>
<p><img alt="pcb_double_sided" src="../../images/week07/Double_Sided/Double_Sided_Schematic.jpg" /></p>
<h3 id="kicad-pcb-editor_1">KiCad PCB Editor<a class="headerlink" href="#kicad-pcb-editor_1" title="Permanent link">&para;</a></h3>
<p>This is where I had to do some research on double sided PCBs with KiCad. I learned that to make a double sided PCB I needed a <em>Via</em> or a connector between layers of a PCB since my PCB was only two layers I could simply use a through hole via I used this image below on figuring out which type of Via I needed</p>
<p><img alt="Type_of_via" src="../../images/week07/Double_Sided/Types_of_Vias.jpeg" /></p>
<p>Next I had to figue out exactly how to encoporate this into KiCad. To figure that out, I went to Youtube and found a couple of useful videos. The first one is from <a href="https://www.youtube.com/@aaltofablab3504">Aalto Fablab</a> and the second is from <a href="https://www.youtube.com/@PhilsLab">Phil&rsquo;s Lab</a>. I recommend watching both videos fully as they are very helpful. Aalto Fablab&rsquo;s video explains how to encoporate a Via into KiCad. Phil&rsquo;s video goes into more depth about adding a copper pour, a large area of copper that can act as a ground plane in some cases, skip to <a href="https://youtu.be/aVUqaB0IMh4?t=5245">1:27:25</a> to see him add the Vias. I was confused initially about copper pours so I used this <a href="https://madpcb.com/glossary/copper-pour/">link</a> to help explain it along with this image.</p>
<p><a href="../../images/week07/Double_Sided/Copper_Pour_example.png">copper pour example</a></p>
<p>Here are the videos I mentioned earlier</p>
<iframe width="560" height="315" src="https://www.youtube.com/embed/ZbP9FEIVfaw?start=4936" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" allowfullscreen></iframe>

<h3 id="copper-pour">Copper Pour<a class="headerlink" href="#copper-pour" title="Permanent link">&para;</a></h3>
<p>For my PCB I needed a copper pour and one trace on the back of the PCB. For the pour I followed the steps in the video above. First I clicked <strong>Add filled zone</strong> and then proceeded to click on a starting point to trace the whole outline. </p>
<p><img alt="copper zone" src="../../images/week07/Double_Sided/Add_Filled_Zone.jpg" /></p>
<p>After that I was brought up with <strong>copper zone properties</strong>. I made sure to select <em>GND</em> as the traces and <em>B.Cu</em> </p>
<p><img alt="image" src="../../images/week07/Double_Sided/Copper_Zone_Properties.jpg" /></p>
<p>Then I traced an outline of the whole design.</p>
<p><img alt="image" src="../../images/week07/Double_Sided/Copper_Zone_Outline.jpg" /></p>
<p>Once I was finished with that <strong>I pressed &ldquo;B&rdquo;</strong> to fill the zone and I was finished with the copper pour.</p>
<p><img alt="image" src="../images/week07/Double_Sided/Copper_zone_filled.jpg" /></p>
<p>This got pretty annoying to design around so I clicked <strong>Show only zone boundaries</strong> to hide the filled zone</p>
<p><img alt="image" src="../../images/week07/Double_Sided/Hidding_Copper_Zone.jpg" /></p>
<h3 id="adding-vias">Adding Vias<a class="headerlink" href="#adding-vias" title="Permanent link">&para;</a></h3>
<p>The next thing I had to do was connect all of the ground pins to the back zone using a via. First I clicked on a trace on the F.Cu like normal then rooted a small trace outward, then I <strong>pressed &ldquo;V&rdquo; to insert a via</strong>. With the via connected I double clicked it and the via was set.</p>
<p><img alt="image" src="../../images/week07/Double_Sided/Adding_Via.jpg" /></p>
<p><img alt="image" src="../../images/week07/Double_Sided/Finished_Via.jpg" /></p>
<p>I had to make sure that the via settings were correct with the ones we had in the lab so I double clicked on the via to make sure I had the right settings. The via hole is the actual size of the rivet and I&rsquo;m using .8 mm rivets. The via diameter should be twice the size of the via hole.</p>
<ul>
<li>Via Hole: 0.8 mm</li>
<li>Via Diameter: 1.6 mm</li>
</ul>
<p>Next I had to connect a VCC trace to another side of the board using a via so similar to the steps above I rooted a trace then added a via, but this time instead of double clicking I created a trace on the B.Cu then moved it toward its intended location. </p>
<p><img alt="image" src="../../images/week07/Double_Sided/Rooting_out_back_trace.jpg" /></p>
<p>Then I added another via to bring me back to the F.Cu then connected that trace to its intended area. </p>
<p><img alt="image" src="../../images/week07/Double_Sided/Adding_back_via.jpg" /></p>
<p><img alt="image" src="../../images/week07/Double_Sided/Back_trace_fully_connected.jpg" /></p>
<p><strong>IMPORTANT</strong></p>
<p>Since I added a pour earlier I had <strong>press &ldquo;B&rdquo;</strong>  to fill the zone again but this time with the new trace I created on the back layer. If I didn&rsquo;t do this, then the trace wouldn&rsquo;t show up on the B.Cu and it would just be the vias.</p>
<p><img alt="image" src="../../images/week07/Double_Sided/Copper_zone_with_trace.jpg" /></p>
<h3 id="exporting_1">Exporting<a class="headerlink" href="#exporting_1" title="Permanent link">&para;</a></h3>
<p>Similar to the documentation for the single sidded PCB I ran the DRC and everything checked out ok. Then I hit Plot and I checked the following</p>
<ul>
<li>F.Cu</li>
<li>B.Cu</li>
<li>Edge.Cuts</li>
</ul>
<p><img alt="image" src="../../images/week07/Double_Sided/Plot_double_sided_PCB.jpg" /></p>
<p>After that I clicked plot</p>
<p>Since I had vias I had to also click <strong>generate drill files</strong> on the same menu. After clicking that I made sure to check the following</p>
<ul>
<li>PTH and NPTH in single file</li>
<li>Drill Units: Millimeters</li>
</ul>
<p><img alt="image" src="../../images/week07/Double_Sided/Generate_drill_files.jpg" /></p>
<p>Then I hit generate drill files. I saw a .drl file in the project directory and I was done exporting.</p>
<h3 id="milling_1">Milling<a class="headerlink" href="#milling_1" title="Permanent link">&para;</a></h3>
<p>For this step I used my <a href="../week05/">PCB Production</a> documentation on double sided PCBs and after that I was done with designing aspect of the PCB.</p>
<h2 id="problems-i-ran-into">Problems I Ran Into<a class="headerlink" href="#problems-i-ran-into" title="Permanent link">&para;</a></h2>
<ul>
<li>Finding the correct symbols with footprints</li>
<li>Creating constraints</li>
<li>Wiring the PCB so the wires don’t cross</li>
<li>Editing the pins on the microcontroller so the wiring in the PCB editor wouldn’t cross over.</li>
</ul>
<h2 id="what-i-learned">What I learned<a class="headerlink" href="#what-i-learned" title="Permanent link">&para;</a></h2>
<p>This week I learned a lot about pull up and pull-down resistors. The microcontroller that we use <a href="https://www.bing.com/search?q=atitny412&amp;cvid=6c44eb7919a044ff808d18c5ae5dc904&amp;aqs=edge..69i57j0l4j69i59j0l3.4154j0j1&amp;pglt=2211&amp;FORM=ANNTA1&amp;PC=W037">ATTiny412</a> has a built-in resistor that when called upon can act as a pull up resistor. Because of this I didn’t have to include a resistor in our designs. I also learned that adding symbols in the schematic editor doesn’t really correlate to anything unless it has a footprint assigned to it. The importance of having the correct wires connected in the schematic editor is critical for the PCB editor. This is because the PCB editor won’t allow you to connect any wires unless they are connected in the schematic editor. In the PCB editor it doesn’t tell you how to connect the wires, but where. Because of this I almost connected my power pin directly into the microcontroller and then into the capacitor (which would make the capacitor useless). I realized this and was careful that the wires were connected as intended in the schematic editor</p>
<h2 id="group-assignment">Group Assignment<a class="headerlink" href="#group-assignment" title="Permanent link">&para;</a></h2>
<p>The group assignment this week was to learn how to operate an oscilloscope and a multimeter. I have used a multimeter before, so the equipment was very familiar, but the oscilloscope was new to me. We were able to judge the analog signal of the LED in the programmers. Unlike a multimeter, which provides an average value when using, an oscilloscope provides real-time adjustments to the LED. The site is located <a href="https://fabacademy.org/2022/labs/charlotte/assignments/week07/">here</a>.</p>
