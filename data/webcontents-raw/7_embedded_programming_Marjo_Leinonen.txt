Program your board to do something, with as many different programming languages and programming environments as possible.  </p>
<h2 id="group-assignment_1">Group assignment<a class="headerlink" href="#group-assignment_1" title="Permanent link">&para;</a></h2>
<p>We worked together with whole Oulu Academy group.  </p>
<p>We tested STM32 Nucleo-64 development board with STM32F401RE MCU and ESP32 board using Arduino IDE.  </p>
<p><img alt="" src="../../images/week09/Nucleo.jpg" />     </p>
<p><img alt="" src="../../images/week09/board1.jpg" />      </p>
<p>We tested the STM32 Nucleo-64 development board with STM32F401RE MCU and ESP32 LOLIN32 development board using Arduino IDE. ST32 is a family of 32-bit microcontroller integrated circuits by STMicroelectronics. This group is related to same 32 bit ARM processor. The STM32 F4-series is the first group of STM32 microcontrollers based on the ARM Cortex-M4F core. The F4-series is also the first STM32 series to have DSP and floating point instructions. The F4 is pin-to-pin compatible with the STM32 F2-series and adds higher clock speed, 64 KB CCM static RAM, full duplex, improved real-time clock, and faster ADCs. All devices offer one 12-bit ADC, a low-power RTC, six general-purpose 16-bit timers including one PWM timer for motor control, two general-purpose 32-bit timers. They also feature standard and advanced communication interfaces.  </p>
<ul>
<li>Up to three I2Cs</li>
<li>Up to four SPIs</li>
<li>Two full duplex I2Ss. To achieve audio class accuracy, the I2S peripherals can be clocked via a dedicated internal audio PLL or via an external clock to allow synchronization.</li>
<li>Three USARTs</li>
<li>SDIO interface</li>
<li>USB 2.0 OTG full speed interface</li>
</ul>
<p>See the <a href="https://www.st.com/resource/en/datasheet/stm32f401re.pdf">data sheet STM32F401x</a> for more information.</p>
<p>ESP32 is a series of low-cost, low-power system on a chip microcontrollers with integrated Wi-Fi and dual-mode Bluetooth. The ESP32 series employs a Tensilica Xtensa LX6 microprocessor in both dual-core and single-core variations and includes in-built antenna switches, RF balun, power amplifier, low-noise receive amplifier, filters, and power-management modules. ESP32 is created and developed by Espressif Systems, a Shanghai-based Chinese company, and is manufactured by TSMC using their 40 nm process. See the more details about <a href="https://wiki.wemos.cc/products:lolin32:lolin32#tutorials">ESP32 LOLIN32</a> in the link.  </p>
<p>We installed Board Manager for the Nucleo board  </p>
<p><img alt="" src="../../images/week09/test1.jpg" />     </p>
<p>Then we defined the board information (Board and Board number) in Tools</p>
<p><img alt="" src="../../images/week09/test3.jpg" />     </p>
<p><img alt="" src="../../images/week09/test4.jpg" />     </p>
<p>and run the blinking test code for the board  </p>
<p><img alt="" src="../../images/week09/test5.jpg" />     </p>
<p><a href="https://gitlab.fabcloud.org/academany/fabacademy/2019/labs/oulu/students/marjo-leinonen/blob/master/docs/images/week09/smtblink.mp4">Video of the blinking</a>  </p>

<h2 id="individual-assignment_1">Individual assignment<a class="headerlink" href="#individual-assignment_1" title="Permanent link">&para;</a></h2>
<h4 id="read-attiny-datasheet">Read ATTiny datasheet<a class="headerlink" href="#read-attiny-datasheet" title="Permanent link">&para;</a></h4>
<p><a href="http://ww1.microchip.com/downloads/en/DeviceDoc/doc8006.pdf">ATTiny datasheet</a> has 238 pages. I scanned it briefly to understand what it contains, so that I can come back to it when needed. I read more detailed the <a href="http://ww1.microchip.com/downloads/en/DeviceDoc/8183S.pdf">short version of the datasheet</a> (22 pages) that gives a quick reference on the microprosessor features, lists Registers and Commands.  </p>
<p>ATtiny24A/44A/84A is a low-power CMOS 8-bit microcontroller based on the AVR enhanced RISC architecture. By executing powerful in structions in a single clock cycle, the ATtiny24A/44A/84A achieves throughputs up to 1 MIPS per MHz allowing the system designer to optimize power consumption versus processing speed.    </p>
<p>Figure 1-1. shows the Pinout of ATtiny24A/44A/84A and Block Diagram (Figure 2-1.) gives an overview of the architechture. More detailed Pin descriptions are in chapter 1.1    </p>
<p>Resources like drivers, application notes,  data sheets and descriptions on development tools are available for download at http://www.atmel.com/avr.  </p>
<p>The documentation contains simple code examples that briefly show how to use various parts of the device. Examples are written in c-code.  </p>
<h4 id="programming-my-board">Programming my board<a class="headerlink" href="#programming-my-board" title="Permanent link">&para;</a></h4>
<p>First I installed <a href="https://www.arduino.cc/en/Main/Software">Arduino 1.8.8</a>.  </p>
<p>Then I copied Boards Manager installation url from <a href="https://github.com/SpenceKonde/ATTinyCore/blob/master/Installation.md">GitHub SpeceKonde</a>. I opened Arduino File/Preferences and pasted the url in Additional Boards Manager URLs.  </p>
<p><img alt="" src="../../images/week09/arduino1.jpg" />  </p>
<p>Then I opened Boards Manager and installed ATTinyCore by SpenceKonde.  </p>
<p><img alt="" src="../../images/week09/arduino2.jpg" />  </p>
<p>Next I opened Tools and defined Board and Chip to ATTiny44, Clock to 20mHz and Programmer as USBTinyISP.  </p>
<p><img alt="" src="../../images/week09/arduino3.jpg" />  </p>
<p>Then I did Burn Bootloader. When using a chip for the first time, or after changing the clock speed or BOD settings, you must do &ldquo;burn bootloader&rdquo; to set the fuses.  </p>
<p>I attached my board to my laptop with FTDI cable to provide the power. Then I inserted my USB programmer to my laptop and connected the ISP cable between the programmer and the board.    </p>
<p><img alt="" src="../../images/week09/arduino4.jpg" />  </p>
<p>I opened Button code in File / Examples / Digital. I set the pin numbers according to my board pins, I have PA7 and PB2 as my button and led ports in ATTiny44A.   </p>
<p>const int buttonPin = 7;     // the number of the pushbutton pin<br />
const int ledPin =  8;      // the number of the LED pin  </p>
<p>I checked the correct pin numbers from <a href="https://github.com/SpenceKonde/ATTinyCore">ATTiny Pinout map</a>  </p>
<p><img alt="" src="../../images/week09/attiny pinout.jpg" />  </p>

<p><img alt="" src="../../images/week09/arduino7.jpg" />  </p>
<p>I checked all the settings again and saw that all the changes I made to ATTiny in Tools were lost. I did the changes again (Board and Chip to ATTiny44, Clock to 20mHz and Programmer as USBTinyISP).  </p>
<p>I saved and ran the code again. Now led was turned on. Led will be off when pushing the button.    </p>
<p><img alt="" src="../../images/week09/arduino5.jpg" />  </p>
<p>In the code, if the buttonState is HIGH, LED is turned on, else LED is turned off.   </p>
<p>if (buttonState == HIGH) {<br />
    // turn LED on:<br />
    digitalWrite(ledPin, HIGH);<br />
  } else {<br />
    // turn LED off:<br />
    digitalWrite(ledPin, LOW);  </p>
<p>I modified the code so that if buttonState is HIGH, LED is turned off.   </p>
<p>if (buttonState == HIGH) {<br />
    // turn LED off:<br />
    digitalWrite(ledPin, LOW);<br />
  } else {<br />
    // turn LED on:<br />
    digitalWrite(ledPin, HIGH);  </p>
<p>Now the led is on only when the button is pushed.  </p>
